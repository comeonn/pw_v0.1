# 三角洲游戏陪玩 H5（老板端 + 打手端）

基于 **Vue 3 + Vite + TypeScript** 的微信服务号 H5 项目，用于三角洲类游戏的陪玩下单、抽奖和打手接单。

## 技术栈

- 前端构建：Vite 7
- 框架：Vue 3 (`<script setup>` + TS)
- 路由：vue-router 4
- UI：纯 CSS，针对微信内 H5（移动端）优化

## 运行与构建

```bash
cd D:\b_project\pw\v0.1

# 开发模式
npm run dev

# 生产构建
npm run build

# 预览构建产物
npm run preview
```

## 微信服务号入口设计

服务号推荐配置两个菜单/按钮，对应两个端：

- **老板端入口**：指向 `/#/`（首页）
- **打手端入口**：指向 `/#/worker/home`

后续接入微信网页授权后，可在后端根据 openid 区分角色（老板 / 打手），并在接口层控制权限与价格。

## 老板端页面（Boss）

- `/`：**首页**
  - 顶部搜索 + Banner（怪兽电竞图）
  - 公告条
  - 中部两个入口卡片：热门推荐、抽奖转盘
  - 下方商品列表（使用 `src/mock/goods.ts` 的数据）
- `/category`：**分类页**
  - 一级分类：护航 / 跑刀 / 33 / 代肝
  - 二级筛选：道具单 / 单局赌单 / 特殊玩法 / 基础保底 / 开业福利
  - 右侧为商品卡片，点击跳转详情
- `/lottery`：**付费抽奖页**
  - 付费抽一次，必得 1 单，最高 3 单
  - 当前用 mock 支付与开奖逻辑（后续可换成真实接口）
- `/goods/:id`：**商品详情**
  - 轮播展示图、标题、简介、价格、销量
  - 底部固定 “立即下单” 按钮（目前跳到 `我的`，可对接真实下单流程）
- `/mine`：**我的**
  - 模拟老板用户卡片（含等级进度条）
  - “我的订单”三个状态入口：待付款 / 待结单 / 已完成
  - 功能入口：我的权益、联系客服

底部 TabBar（三个按钮）：

- 首页 `/`
- 分类 `/category`
- 我的 `/mine`

## 打手端页面（Worker）

打手端所有页面底部使用打手专属导航，而不是老板 TabBar。

### `/worker/home` —— 接单大厅

- 头部展示打手档案（`MOCK_WORKER_PROFILE`）：
  - 金牌/普通标识、ID、今日/本月收入、完单数、评分
  - “钱包明细 ›” 快捷入口
- 接单大厅列表（`MOCK_AVAILABLE_ORDERS`）：
  - 单子标题、玩法模式（护航/跑刀/33/代肝）
  - **打手价**（结算价）与 **老板价**（参考价）同时展示
  - 预计耗时、创建时间、老板备注
  - 操作按钮：忽略 / 接单（目前仅 UI，后续可接真实状态流）

### `/worker/orders` —— 我的单

- 顶部状态切换：
  - 全部 / 进行中 / 待确认 / 已完成
- 列表来源 `MOCK_MY_ORDERS`：
  - 显示订单号、标题、模式、打手价、时间
  - 状态标签颜色区分：进行中 / 待老板确认 / 已完成

### `/worker/wallet` —— 钱包

- 余额卡片（`MOCK_WALLET`）：
  - 可提现余额
  - 累计收入、累计提现
  - “提现”按钮
- 账单明细（`MOCK_WALLET_TXN`）：
  - 收入、提现申请等记录，正负金额、时间、备注
- 提现弹窗：
  - 说明目前通过 **添加客服微信** 完成提现
  - 展示客服微信号和二维码占位（可替换为真实二维码）

### 打手端导航

底部 `worker-nav`：

- 大厅 `/worker/home`
- 我的单 `/worker/orders`
- 钱包 `/worker/wallet`

## Mock 数据位置

- `src/mock/goods.ts`：商品信息（老板端使用）
- `src/mock/worker.ts`：打手信息、可接订单、我的订单、钱包数据（打手端使用）

后续接后端接口时，可以直接替换这两个 mock 模块的使用方式。
