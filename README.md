# 三角洲游戏陪玩 H5（老板端 + 打手端 + 运营后台）

基于 **Vue 3 + Vite + TypeScript** 的微信服务号 H5 项目，用于三角洲类游戏的陪玩下单、抽奖和打手接单；另含 **运营后台**（PC 端），供运营人员管理用户、订单、商品与文案。

## 技术栈

- 前端构建：Vite 7
- 框架：Vue 3 (`<script setup>` + TS)
- 路由：vue-router 4
- UI：纯 CSS；H5 端针对微信内移动端优化，运营后台为 PC 端（简洁、偏运营工具风）

## 微信服务号入口设计

服务号推荐配置两个菜单/按钮，对应两个端：

- **老板端入口**：指向 `/#/`（首页）
- **打手端入口**：指向 `/#/worker/home`

后续接入微信网页授权后，可在后端根据 openid 区分角色（老板 / 打手），并在接口层控制权限与价格。

## 老板端页面（Boss）

- `/`：**首页**
  - 顶部 Banner、公告条、搜索框可配置（文案管理可上传首页海报，公告为文案配置）
  - 中部两个入口卡片：热门推荐、抽奖转盘（海报可由文案管理上传替换）
  - 下方商品列表（使用 `src/mock/goods.ts` 的数据）；商品卡展示老板价、已售，无划线优惠。
- `/category`：**分类页**
  - 一级分类：护航 / 跑刀 / 33 / 代肝
  - 二级筛选：道具单 / 单局赌单 / 特殊玩法 / 基础保底 / 开业福利
  - 二级分类条支持左右滑动切换
  - 右侧为商品大图卡片，点击跳转详情
- `/lottery`：**付费抽奖页**
  - 转盘三档奖品文案可配置；右上角「公示」文字按钮，点击弹窗展示管理端设置的公示文本。
  - 当前用 mock 支付与开奖逻辑（后续可换成真实接口）
- `/goods/:id`：**商品详情**
  - 轮播展示图、标题、简介、价格、销量
  - 底部固定 “立即下单” 按钮（目前跳到 `我的`，可对接真实下单流程）
- `/mine`：**我的**
  - 模拟老板用户卡片（含等级进度条）
  - “我的订单”四个状态入口：待付款、待派单、待结单、已完成；点击任一项进入订单列表页 `/mine/unassigned`，顶部 Tab 可切换四类，待派单下列表可点击「派单」填写手机号、游戏 ID、姓名、备注后提交。
  - 功能入口：我的权益、联系客服

底部 TabBar（三个按钮）：

- 首页 `/`
- 分类 `/category`
- 我的 `/mine`

## 打手端页面（Worker）

打手端所有页面底部使用打手专属导航，而不是老板 TabBar。

### `/worker/orders` —— 我的单

- 顶部状态切换：

### `/worker/wallet` —— 钱包

## 运营后台（Admin，PC 端）

仅运营人员使用，布局：**左侧菜单 + 顶部导航 + 右侧内容区**，适合 PC 端使用。

- **登录**：`/admin/login`（账号密码，后续对接后端鉴权）
- **用户管理**：`/admin/users` — 老板列表 / 打手列表（搜索、封禁、查看详情）；**设置保护期时间**：老板下单后金牌打手可立即在接单大厅看到该单，普通打手需过了保护期后才可见；弹窗内可配置保护期数值及单位（秒 / 分钟 / 小时），对接后端生效。
- **订单管理**：`/admin/orders` — 订单列表（按状态筛选）；商品列展示商品图片与名称；操作列提供「详情」按钮，点击弹窗展示该订单商品详情（商品图、名称、金额、订单号、下单时间）。
- **抽奖管理**：`/admin/lottery` — 设置老板端付费抽奖：是否开启、抽奖单价、奖品设置（选择商品与抽中概率，概率合计需为 100%）、活动商品名称与封面图、规则说明、**公示设置**（文本展示在老板端抽奖页右上角「公示」按钮弹窗中）；对接后端后在此配置并发布。
- **提现管理**：`/admin/withdraws` — 打手提现申请列表，标记「已线下转账」（线下通过运营微信转账）。当前在侧栏中隐藏，路由与页面保留，后续可恢复展示。
- **基础配置**
  - **商品管理**：`/admin/goods` — 商品增删改；列表展示商品 ID、名称、分类、**老板价格**、**打手金额**、销量及操作（编辑、删除）；已去掉状态、排序、下架列。
  - **文案管理**：`/admin/copy` — **首页海报**、热门推荐海报、抽奖转盘海报上传（首页与两个入口卡片可展示管理员上传的图片）；首页公告、**客服微信号**（文本）、**客服微信号图片**（上传二维码/图片）；对接后端后在此编辑并发布。

访问入口：浏览器打开 `http://localhost:5173/admin` 会跳转到 `/admin/users`，未登录时可先访问 `/admin/login`。后台与 H5 共用同一套构建与部署，部署后后台地址为 `https://你的域名/admin`。

## 微信登录 / 下单 / 支付接入 Checklist（概要）

> 正式接入建议等：**域名备案完成 + HTTPS 配好 + 服务器上线** 后再与后端一起联调。  
> 以下只列前后端需要准备的关键点，方便后续按项推进。

1. **基础设施**
   - 购买云服务器（推荐 Linux + 宝塔 + Nginx）
   - 绑定并备案域名，开启 HTTPS（Let’s Encrypt 即可）
   - 使用 `npm run build` 构建前端，将 `dist` 部署到 Nginx，并配置：
     - `try_files $uri $uri/ /index.html;` 支持前端路由
2. **微信公众号 / 微信开放平台配置**
   - 申请并配置服务号 / 订阅号（推荐服务号）
   - 在公众号后台设置：
     - **网页授权回调域名**（用于微信登录）
     - **JS 接口安全域名**（用于 JS-SDK / H5 支付）
   - 准备好 AppID、AppSecret（不直接暴露给前端）
3. **微信登录（网页授权）对接**
   - 约定后端回调地址，如：`/api/wechat/oauth/callback`
   - 前端在需要登录的位置跳转到后端提供的登录链接（后端负责拼装 OAuth URL）
   - 后端在回调中拿到 `code` → 换取 `openid` / `unionid` / `access_token`
   - 后端生成自己的用户 Token 返回前端；前端存到 localStorage / cookie
   - 在 axios 等请求工具中统一带上 Token
4. **下单流程对接**
   - 约定下单接口，例如：
     - `POST /api/orders`：创建订单（传入商品 ID、数量、角色信息等）
     - `GET /api/orders/:id`：查询订单详情
     - `GET /api/orders/my`：我的订单列表（按角色区分老板 / 打手）
   - 前端在：
     - 老板端 `GoodsDetail` 的“立即下单”
     - 打手端接单大厅的“接单”
     中，调用对应接口并处理返回状态
5. **微信支付对接（统一由后端与微信商户侧打交道）**
   - 后端持有微信商户号、API Key、证书，前端只负责：
     - 调用 `POST /api/pay/unified` 获取支付参数
     - 在微信内 H5 调用 `WeixinJSBridge` 或 JS-SDK 发起支付
   - 后端实现支付回调通知（如 `/api/pay/notify`），更新订单状态
6. **打手端业务接口**
   - 接单：`POST /api/worker/orders/accept`
   - 订单状态流转：进行中 / 待确认 / 已完成
   - 钱包：
     - 收入结算记录
     - 可提现余额查询
     - 提现申请接口（与客服 / 微信转账结合）

当前仓库的前端已用 mock 数据把老板端 + 打手端完整走通，后续只需要按以上 Checklist 替换对应的 mock 接口调用即可。
